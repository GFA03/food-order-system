# Functional Requirements
## Project: OmniEats - AI-Powered Microservices Food Marketplace

The application allows user registration and account creation.

The application allows user authentication and access based on JWT tokens.

The application allows role-based differentiation of users (standard USER and ADMIN).

The application allows users to manage their profiles, including delivery coordinates and dietary preferences.

The application allows browsing and searching restaurants by cuisine tags.

The application allows viewing restaurant menus and menu items.

The application allows placing orders with multiple items.

The application allows tracking order status and history.

The application allows AI-driven semantic prompting for menu discovery (e.g., "I want something vegan and spicy").

The application allows pagination and sorting of restaurants and menu items.

The application allows admins to manage restaurants, menu items, and cuisine tags.

The application allows CRUD operations on all domain entities via RESTful APIs.

The application allows data validation on client and server sides.

The application allows graceful error handling for various fault states.

The application allows asynchronous status updates for orders.

The application allows persistent cart management.

The application allows checkout execution and order instantiation.
